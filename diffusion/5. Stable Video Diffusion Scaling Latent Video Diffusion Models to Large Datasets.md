# Stable Video Diffusion: Scaling Latent Video Diffusion Models to Large Datasets 阅读笔记

论文：Stable Video Diffusion: Scaling Latent Video Diffusion Models to Large Datasets  
链接：https://arxiv.org/abs/2311.15127  
PDF：https://arxiv.org/pdf/2311.15127  
作者：Andreas Blattmann, Tim Dockhorn, Sumith Kulal, Daniel Mendelevitch, Maciej Kilian, Dominik Lorenz, Yam Levi, Zion English, Vikram Voleti, Adam Letts, Varun Jampani, Robin Rombach  
常用简称：SVD，Stable Video Diffusion

## 1. 阅读计划回顾

这篇论文不要先读成“又一个 text-to-video 模型”。更好的入口是：

```text
图像扩散模型已经很强
  -> 直接插 temporal layer 能做视频
  -> 但视频质量真正卡在数据分布、训练阶段和噪声设定
  -> 所以论文固定相对简单的视频 LDM 架构
  -> 系统研究数据策展和三阶段训练
  -> 最后证明同一个 base model 可以迁移到 T2V、I2V、插帧、多视角生成
```

建议按以下顺序阅读：

1. `Abstract`、`Introduction`：确认论文的主张不是“新架构”，而是“视频 LDM 的训练和数据策略”。
2. `Section 3.1`：读懂 LVD/LVD-F 数据从原始长视频到可训练 clip 的处理流程。
3. `Section 3.2-3.4`：读三阶段训练，重点看每个阶段被消融证明了什么。
4. `Section 4.1` 和 `Appendix D.1-D.2`：读 EDM 公式、噪声分布迁移、base model 训练细节。
5. `Section 4.2-4.5`：看 text-to-video、image-to-video、frame interpolation、multi-view generation 如何从同一个 motion prior 派生。
6. `Appendix E`：把 human preference、Elo、UCF101 FVD、SOTA 对比当作因果证据，而不是只抄指标。
7. `Appendix A`：回到短视频、采样速度、VRAM、运动不足和安全风险。

## 2. 阅读主线

显式问题是：

```text
如何训练一个高分辨率、效果强的 latent video diffusion model，
使它既能做 text-to-video，也能迁移到 image-to-video 和 multi-view generation。
```

真正的结构张力是：

```text
视频生成表面上像一个架构问题：
  如何加 temporal convolution？
  如何加 temporal attention？
  如何保持帧间一致？

但论文认为更大的瓶颈在训练分布：
  原始视频有转场、淡入淡出、静态片段、低美学质量、大量文字、水印、caption 不准。
  如果把这些直接喂给扩散模型，模型会忠实学习这些坏模式。
```

第一原则是：

```text
视频扩散模型不是只学“图像内容 + 时间层”。
它必须从数据中学到一个可泛化的 motion prior。

motion prior 的质量由三件事共同决定：
  图像先验是否强；
  视频预训练数据是否足够大且足够干净；
  高质量微调是否保留并放大这个先验。
```

所以 SVD 的核心判断是：

```text
先用 Stable Diffusion 2.1 提供空间视觉先验，
再用大规模 curated video clips 学运动先验，
最后用小规模高质量数据对齐到高分辨率任务。
```

## 3. 论文要解决的问题

在 SVD 之前，很多 video LDM 的路径是：

```text
pretrained text-to-image model
  -> 插入 temporal layers
  -> 在较小的高质量视频数据上 finetune
```

这条路径能工作，但有一个隐藏问题：如果只在小数据上微调，模型学到的运动分布很窄；如果直接用大规模网络视频，模型又会学习到剪辑、静止镜头、字幕、水印、低质量画面等坏模式。

SVD 把问题改写成：

```text
怎样构建一个足够干净的大规模视频预训练集，
并证明这个预训练阶段对最终高质量视频生成仍然有贡献？
```

论文把答案拆成三个训练阶段：

1. `Stage I: image pretraining`：从 Stable Diffusion 2.1 初始化空间层，继承强图像先验。
2. `Stage II: video pretraining`：在大规模 curated video dataset 上学习 motion prior。
3. `Stage III: high-quality video finetuning`：用更小但更高质量的视频数据做高分辨率任务对齐。

## 4. 核心贡献

这篇论文的贡献可以读成四层。

第一层是数据策展流程。论文从大规模原始长视频出发，经过 cut detection、keyframe-aware clipping、optical flow、synthetic captioning、CLIP similarity、aesthetic score、OCR/text area filtering，构建 LVD 和 LVD-F。

第二层是训练范式。论文明确把视频模型训练拆成 image pretraining、video pretraining、high-quality finetuning，并用消融证明每个阶段都有独立价值。

第三层是工程化 SVD base model。论文从 SD 2.1 出发，插入 temporal convolution 和 temporal attention，使用 EDM preconditioning 与更高噪声分布，在 LVD-F 上训练出可迁移的 video base model。

第四层是 motion prior 的泛化性。base model 不只服务于 text-to-video，还能迁移到 image-to-video、camera motion LoRA、frame interpolation、多视角生成。

## 5. 核心符号

```text
x0: 干净数据样本。对视频模型来说，可以理解为一段视频 latent 序列。

x: 加噪后的样本。

c: 条件信号，可以是 text prompt、CLIP image embedding、frame rate、motion score、camera elevation 等。

sigma: 噪声标准差。sigma 越大，样本越接近高斯噪声。

p_data(x0, c): 真实数据和条件的联合分布。

p(x; sigma): 把真实数据加上方差 sigma^2 的高斯噪声后得到的分布。

n: 高斯噪声，n ~ N(0, sigma^2 I)。

D_theta(x; sigma, c): 可学习 denoiser，输入带噪样本、噪声强度和条件，输出干净样本估计。

F_theta: UNet 主体网络。EDM 里 D_theta 不是直接等于 F_theta，而是由 F_theta 和一组 preconditioning 系数组成。

s_theta(x; sigma): score function 的模型估计。

lambda_sigma: 不同噪声强度下的 loss 权重。

w: classifier-free guidance scale。

LVD: Large Video Dataset，论文构建的大规模视频 clip 数据集。

LVD-F: 经过过滤后的 LVD，用于大规模视频预训练。

Stage I: image pretraining。

Stage II: video pretraining。

Stage III: high-quality video finetuning。
```

## 6. 方法框架

SVD 的整体框架可以压缩成：

```text
Stable Diffusion 2.1
  -> 先适配 EDM 风格连续噪声和新 preconditioning
  -> 插入 temporal convolution + temporal attention
  -> 在 LVD-F 上做低分辨率视频预训练
  -> 提升到 320 x 576 继续训练 base model
  -> 根据任务做高分辨率 finetune
      -> text-to-video
      -> image-to-video
      -> frame interpolation
      -> multi-view generation
      -> camera motion LoRA
```

最关键的设计不是某一个 temporal layer，而是这个组合：

```text
强图像先验 + 大规模干净视频预训练 + 高质量任务微调
```

如果只看架构，SVD 很接近 Video LDM：在 2D UNet 的 spatial convolution / attention 后插入 temporal convolution / attention。论文刻意固定这个相对朴素的架构，以便把主要变量放在数据和训练流程上。

## 7. 数据策展：LVD 到 LVD-F

论文先收集大规模长视频，处理后得到 LVD：

```text
LVD:
  clips: 577M / 580M 量级
  total duration: 约 212 年
  mean clip duration: 约 11.58 秒
```

经过过滤后得到 LVD-F：

```text
LVD-F:
  clips: 152M
  total duration: 约 50.64 年
  mean clip duration: 约 10.53 秒
```

数据处理流程：

```text
raw long videos
  -> cascaded cut detection
  -> keyframe-aware clipping
  -> optical flow annotation
  -> synthetic captioning
      -> CoCa caption
      -> VideoBLIP caption
      -> LLM summary caption
  -> CLIP text-image similarity
  -> aesthetic score
  -> OCR / text area detection
  -> filtering thresholds
  -> LVD-F
```

### 7.1 为什么要 cut detection

如果训练数据里有剪辑点、淡入淡出、场景切换，扩散模型会把这些当作真实视频运动的一部分，生成视频时也可能出现不自然跳切。

论文使用三层 cascaded cut detector，在不同 FPS 和阈值下检测突然切换和缓慢转场。这个设计的目的不是追求语义理解，而是先保证训练 clip 在时间上连续。

### 7.2 为什么要 optical flow

原始视频里有大量近似静态片段。对视频生成来说，静态片段太多会让模型倾向于生成“几乎不动”的视频。

论文先用 Farneback optical flow 在 2 FPS 下估计粗略 motion score，用来过滤静态视频。对更小的高质量微调数据，论文使用更准确的 RAFT flow，因为成本可控。

### 7.3 为什么要三个 caption

论文使用三种自动 caption：

```text
CoCa:
  图像 captioner，擅长描述空间内容。

VideoBLIP:
  视频 captioner，更容易捕捉动作和时间信息。

LLM summary:
  结合 CoCa 与 VideoBLIP，得到更自然的综合描述。
```

消融结果显示，CoCa caption surprisingly useful。最终训练时不是只用 CoCa，而是按比例采样：

```text
p_CoCa = 0.5
p_V-BLIP = 0.25
p_LLM = 0.25
```

### 7.4 过滤阈值

论文在 LVD-10M 上对不同 annotation 轴分别做过滤消融，然后用 human preference / Elo 选阈值。

最终倾向的过滤策略包括：

```text
motion:
  过滤掉最静态的 25%。

aesthetic:
  过滤掉 aesthetics score 最低的 25%。

CLIP similarity:
  过滤掉 CLIP text-image similarity 最低的 50%。

text area:
  过滤掉文字区域最大的 25%。
  最终构建 LVD-F 时，还用 text detector 过滤掉检测文字面积超过 7% 的 clip。
```

这里的关键不是阈值本身，而是方法论：

```text
不要预设哪个过滤器有用。
对每个过滤轴训练同构模型，再让人类偏好决定过滤策略是否改善生成质量。
```

## 8. 关键公式与推导

### 8.1 连续时间扩散视角

论文在附录中使用 continuous-time diffusion / EDM 框架。定义：

```text
p_data(x0): 真实数据分布
p(x; sigma): x0 加上方差 sigma^2 的高斯噪声后的分布
```

当 `sigma_max` 足够大时：

```text
p(x; sigma_max) approx N(0, sigma_max^2 I)
```

所以采样可以从高斯噪声开始：

```text
x_M ~ N(0, sigma_max^2 I)
```

再沿着 probability flow ODE 逐步去噪：

```text
dx = - dot_sigma(t) * sigma(t) * grad_x log p(x; sigma(t)) dt
```

### 推导

扩散模型需要知道每个噪声水平下，样本应该往哪个方向移动才更像真实数据。这个方向由 score function 给出：

```text
grad_x log p(x; sigma)
```

EDM 常把 score 写成 denoiser 的形式：

```text
s_theta(x; sigma) = (D_theta(x; sigma) - x) / sigma^2
```

这个式子说明：

```text
D_theta(x; sigma) - x
```

就是 denoiser 认为当前带噪样本应该移动的方向；除以 `sigma^2` 后变成 score 的尺度。

### 直觉

模型不是直接学“生成视频”，而是在每个噪声强度下学：

```text
给我一个被噪声破坏的视频 latent，
我估计它原本干净的样子。
```

采样时反复调用这个能力，就能从纯噪声走回视频分布。

### 8.2 Denoising Score Matching 目标

论文使用的训练目标可以写成：

```text
E_{(x0,c) ~ p_data, (sigma,n) ~ p(sigma,n)}
[
  lambda_sigma * || D_theta(x0 + n; sigma, c) - x0 ||_2^2
]
```

其中：

```text
p(sigma,n) = p(sigma) N(n; 0, sigma^2 I)
```

### 推导

训练时：

```text
1. 从真实数据中采样 x0 和条件 c。
2. 采样一个噪声强度 sigma。
3. 采样高斯噪声 n。
4. 构造带噪输入 x = x0 + n。
5. 让 D_theta(x; sigma, c) 预测 x0。
6. 用加权 MSE 更新模型。
```

这个目标的重点是：模型的监督信号来自真实干净样本 `x0`，而不是来自人工标注的运动标签。

### 直觉

训练阶段知道干净视频是什么，所以可以直接监督 denoiser。推理阶段不知道干净视频，只能从噪声开始，不断用 denoiser 的估计更新样本。

## 9. EDM Preconditioning

SVD 不直接让 UNet 输出干净样本，而是用 EDM preconditioning：

```text
D_theta(x; sigma)
  = c_skip(sigma) * x
    + c_out(sigma) * F_theta(c_in(sigma) * x; c_noise(sigma))
```

其中 `F_theta` 是网络主体。

这可以读成：

```text
denoiser 输出 = 跳连保留的输入 + 网络预测的残差修正
```

论文把 SD 2.1 的噪声设定迁移到 EDM 风格，使用：

```text
c_skip(sigma) = (sigma^2 + 1)^(-1)

c_out(sigma) = -sigma / sqrt(sigma^2 + 1)

c_in(sigma) = 1 / sqrt(sigma^2 + 1)

c_noise(sigma) = 0.25 * log(sigma)
```

并使用：

```text
log sigma ~ N(P_mean, P_std^2)

lambda(sigma) = (1 + sigma^2) * sigma^(-2)
```

### 为什么要改噪声分布

论文指出，原始 SD 2.1 的最大离散噪声水平仍然有比较高的 signal-to-noise ratio。高分辨率视频微调时，如果高噪声区域覆盖不足，模型在强条件或高分辨率场景下容易出现问题。

所以 SVD 在训练阶段多次把噪声分布向更高噪声移动：

```text
base high-res training:
  P_mean = 0

high-res text-to-video:
  P_mean = 0.5, P_std = 1.4

image-to-video base:
  P_mean = 0.7, P_std = 1.6

image-to-video high-res:
  P_mean = 1.0, P_std = 1.6
```

直觉上，条件越强、分辨率越高，越需要模型在更困难的噪声区域保持稳定去噪能力。

## 10. Classifier-Free Guidance

论文使用 classifier-free guidance。公式是：

```text
D^w(x; sigma, c)
  = w * D(x; sigma, c) - (w - 1) * D(x; sigma)
```

等价写法：

```text
D^w = D_uncond + w * (D_cond - D_uncond)
```

其中：

```text
D_cond: 有条件预测
D_uncond: 无条件预测
w: guidance strength
```

### 直觉

`D_cond - D_uncond` 表示条件信号带来的方向差异。`w` 越大，模型越强地朝条件描述靠拢。

但在 image-to-video 里，论文发现固定 guidance scale 有问题：

```text
scale 太小:
  生成视频与条件图不一致。

scale 太大:
  画面过饱和或出现 artifact。
```

因此论文提出沿帧轴线性增大的 guidance：

```text
early frames:
  guidance 小，避免破坏输入图一致性。

later frames:
  guidance 大，允许后续帧更强地跟随生成条件并发展运动。
```

这说明 image-to-video 的约束不是所有帧一样强：越靠近输入图，越应该尊重 conditioning frame；越靠后，越需要生成合理运动。

## 11. 训练过程

### 11.1 Stage I: Image Pretraining

目的：

```text
继承 Stable Diffusion 2.1 的空间视觉先验。
```

消融方式：

```text
同样的视频模型架构
  A: 空间层从 pretrained image model 初始化
  B: 空间层随机初始化
```

结果：

```text
image-pretrained model 在视觉质量和 prompt following 上都更受人类偏好。
```

结论：

```text
视频模型不应该从零开始学“什么是好图像”。
先继承强图像先验，再学习时间维度更有效。
```

### 11.2 Image Model Finetuning 到 EDM

在插入 temporal layers 之前，论文先让 SD 2.1 适配新的 EDM preconditioning 和连续噪声。

流程：

```text
load SD 2.1
replace preconditioning / noise distribution

for first 1k iterations:
    freeze almost all F_theta
    train mainly time-embedding layer
    resolution = 512 x 512

for next 30k iterations:
    train all layers
    resolution = 256 x 384
```

这样做是为了避免一下子破坏 SD 2.1 已有内部表示，同时把时间/噪声接口迁移到新框架。

### 11.3 Stage II: Video Pretraining

架构修改：

```text
start from image UNet
insert temporal convolution and temporal attention
after corresponding spatial convolution and spatial attention layers
```

参数规模：

```text
new temporal parameters: 656M
total UNet parameters: 1521M
```

训练流程：

```text
initialize from EDM-adapted SD 2.1
insert temporal layers

train on LVD-F:
    frames = 14
    resolution = 256 x 384
    iterations = 150k
    batch size = 1536
    optimizer = AdamW
    learning rate = 1e-4
    text dropout = 15%

continue training:
    frames = 14
    resolution = 320 x 576
    iterations = 100k
    batch size = 768
    shift noise distribution to higher noise
```

训练时还加入 micro-conditioning：

```text
frame rate
motion score
```

这让推理时可以一定程度控制生成视频的运动量。

### 11.4 Stage III: High-Quality Finetuning

Stage III 的目标不是重新学习运动，而是把 Stage II 学到的 motion prior 对齐到高质量、高分辨率输出。

论文用于分析的 finetuning 数据大约是：

```text
250K pre-captioned high-fidelity video clips
```

最终高分辨率 text-to-video / image-to-video 使用约：

```text
1M high-quality video samples
```

实验比较三种初始化：

```text
A: image model only，没有 video pretraining
B: uncurated video pretraining
C: curated video pretraining
```

结果：

```text
C > B > A
```

并且 curated pretraining 的优势在高质量微调 50K steps 后仍然保留。

## 12. 推理与下游任务

### 12.1 Text-to-Video

高分辨率 text-to-video 从 base model 微调：

```text
dataset: 约 1M high-quality video samples
resolution: 576 x 1024
iterations: 50k
batch size: 768
learning rate: 3e-5
P_mean = 0.5
P_std = 1.4
EMA decay = 0.9999
```

推理心智模型：

```text
input text prompt
sample initial video latent noise
for each denoising step:
    predict denoised video latent conditioned on text
    apply classifier-free guidance
    update latent sequence
decode video frames
```

### 12.2 Image-to-Video

image-to-video 把文本条件换成图像条件：

```text
text embedding
  -> CLIP image embedding of conditioning frame
```

同时把输入图像作为 concat conditioning：

```text
conditioning image
  -> add small noise
  -> encode with SD 2.1 VAE encoder
  -> copy across time axis
  -> concatenate channel-wise to UNet input
```

论文强调：

```text
不使用 masking techniques。
直接把 conditioning frame 的 latent 复制到所有时间步作为额外输入。
```

训练分两段：

```text
base image-to-video:
    resolution = 320 x 576
    iterations = 50k
    batch size = 768
    learning rate = 3e-5
    P_mean = 0.7
    P_std = 1.6

high-res image-to-video:
    resolution = 576 x 1024
    frames = 14 or 25
    iterations = 50k
    batch size = 768
    learning rate = 3e-5
    P_mean = 1.0
    P_std = 1.6
```

### 12.3 Camera Motion LoRA

为了控制相机运动，论文在 temporal attention blocks 中训练 LoRA：

```text
rank = 16
iterations = 5k
motion-specific datasets:
    horizontal moving
    zooming
    static
```

这说明 SVD 的 temporal layers 中已经包含可调的运动表征。LoRA 不需要重训完整模型，只需在 temporal attention 上添加低秩适配。

### 12.4 Frame Interpolation

插帧模型从 high-resolution text-to-video model 微调。

任务：

```text
given first frame and last frame
predict three middle frames
effectively increase frame rate by 4x
```

输入构造：

```text
z_s: start frame latent
z_e: end frame latent
z_m: learned mask embedding

latent sequence:
    {z_s, z_m, z_m, z_m, z_e}
```

再拼接 binary mask，最终 UNet 输入形状类似：

```text
(5, 9, 52, 128)
```

论文发现只需约 `10k` iterations 就能得到可用模型，这进一步支持 base model 已经学到了强 motion prior。

### 12.5 Multi-View Generation

论文把 multi-view generation 看成一种特殊视频：

```text
对象不动，摄像机绕物体运动。
```

这使得 video diffusion model 很适合迁移到多视角生成，因为 temporal attention 天然帮助多帧一致性。

训练设置：

```text
start from high-resolution image-to-video SVD
input: one object image
output: sequence of novel views
dataset:
    Objaverse subset: 150K curated CC-licensed synthetic objects
    render 360-degree orbital videos, 21 frames
    resolution = 576 x 576
    elevation in [-5°, 30°]

training:
    iterations = 12k
    batch size = 16
    GPUs = 8 x A100 80GB
    learning rate = 1e-5
    time = about 16 hours
```

与 image-prior 和 no-prior 对比：

```text
SVD-MV:
  从视频先验微调

SD2.1-MV:
  从图像先验微调

Scratch-MV:
  随机初始化
```

结果显示：

```text
SVD-MV 从很早的 finetuning step 开始就更好。
```

这说明 SVD 的 motion prior 也可以被解释为一种弱 3D / camera motion prior。

## 13. 实验结论

### 13.1 Image Pretraining 有用

实验问题：

```text
视频模型是否必须从 pretrained image model 初始化？
```

证据：

```text
初始化空间层的模型，在 human preference 中明显优于随机初始化空间层的模型。
```

结论：

```text
图像先验是视频模型的必要地基。
```

### 13.2 Data Curation 比单纯数据量更关键

实验问题：

```text
过滤后数据更少，会不会反而损失性能？
```

证据：

```text
LVD-10M-F 比 LVD-10M 小约 4 倍，
但训练出的模型在人类偏好上优于未过滤 LVD-10M。
```

进一步比较：

```text
LVD-10M-F vs WebVid-10M
LVD-10M-F vs InternVid-10M
```

结果：

```text
LVD-10M-F 尽管更小，仍在 spatiotemporal quality 和 prompt alignment 上更受偏好。
```

结论：

```text
对生成式视频模型来说，坏样本不是中性噪声，而是会被模型学习的生成模式。
```

### 13.3 Curated Data 在大规模下仍有效

实验问题：

```text
小规模上有效的过滤策略，到了 50M 样本规模还有效吗？
```

证据：

```text
50M curated samples 训练出的模型优于 50M uncurated samples。
50M curated samples 也优于 LVD-10M-F。
```

结论：

```text
质量和规模不是二选一。
在足够大规模上，系统过滤仍然能带来收益。
```

### 13.4 Video Pretraining 的收益会保留到 Finetuning 后

实验问题：

```text
高质量 finetuning 会不会抹平 video pretraining 的差异？
```

证据：

```text
finetuning 10K 和 50K steps 后，
curated video pretrained model 仍然排名最高。
```

结论：

```text
Stage II 不是临时 warmup。
它学到的 motion prior 会影响最终模型上限。
```

### 13.5 Base Model 的零样本视频能力

UCF-101 zero-shot text-to-video FVD：

```text
CogVideo (ZH): 751.34
CogVideo (EN): 701.59
Make-A-Video: 367.23
Video LDM: 550.61
MagicVideo: 655.00
PYOCO: 355.20
SVD: 242.02
```

结论：

```text
即使不针对 UCF-101 训练，SVD base model 的 motion prior 也很强。
```

### 13.6 Image-to-Video 优于闭源模型

论文用 human preference 比较 SVD image-to-video 与 Gen-2、PikaLabs。

结论：

```text
SVD 的 25-frame image-to-video model 在视觉质量上更受人类评估者偏好。
```

注意这里主要是 human preference，而不是一个完全标准化、可复现的自动指标。

### 13.7 Multi-View Generation 证明了视频先验的迁移性

GSO test dataset 上的多视角指标：

```text
SyncDreamer:
  LPIPS 0.18, PSNR 15.29, CLIP-S 0.88

Zero123:
  LPIPS 0.18, PSNR 14.87, CLIP-S 0.87

Zero123XL:
  LPIPS 0.20, PSNR 14.51, CLIP-S 0.87

Scratch-MV:
  LPIPS 0.22, PSNR 14.20, CLIP-S 0.76

SD2.1-MV:
  LPIPS 0.18, PSNR 15.06, CLIP-S 0.83

SVD-MV:
  LPIPS 0.14, PSNR 16.83, CLIP-S 0.89
```

结论：

```text
视频先验比纯图像先验更适合多视角一致生成。
```

原因是多视角序列和视频序列共享一个结构：多个帧之间必须保持对象身份和外观一致，同时允许相机或视角变化。

## 14. 本次疑问记录

### Q1：这篇论文的核心是不是新架构？

不是。论文有架构细节，但它的主要贡献是证明：

```text
数据策展 + 三阶段训练
```

对视频扩散质量的影响很大。它刻意使用相对标准的 Video LDM 架构，把研究重心放在数据和训练策略上。

### Q2：为什么不能直接用小规模高质量视频 finetune？

因为小规模高质量数据可以提升视觉质量，但很难覆盖足够丰富的运动分布。SVD 的 Stage II 用大规模 curated videos 学一般运动先验，Stage III 再做高质量对齐。

### Q3：为什么过滤静态视频重要？

如果静态片段太多，模型会学到“视频可以几乎不动”。这会让生成结果看起来像轻微抖动的图像，而不是有真实运动的视频。

### Q4：为什么过滤文字区域？

训练集中大量字幕、水印、UI 文本会让模型生成时也倾向于产生伪文字或不想要的文字块。论文用 OCR/text area filtering 减少这个偏差。

### Q5：为什么 image-to-video 要沿时间增加 guidance？

输入图本身是强条件。前几帧需要保持和输入图一致，过强 guidance 可能导致过饱和；后几帧则需要更强条件推动视频发展。因此 guidance scale 沿帧轴递增。

### Q6：multi-view 结果是否等于模型真的理解 3D？

不能简单等同。更准确地说，SVD 学到的 temporal consistency 和 camera/object motion prior 能迁移到多视角一致生成。它表现出有用的 3D prior，但不是显式几何建模。

## 15. 最小心智模型

用一句话理解 SVD：

```text
SVD = Stable Diffusion 2.1 的图像先验
      + temporal layers
      + 大规模 curated video pretraining
      + 高质量视频 finetuning
      + EDM 高噪声训练设定
```

用训练流程理解：

```text
Stage I:
    先别让视频模型从零学画面。
    用 SD 2.1 给它空间视觉能力。

Stage II:
    再用大规模干净视频让它学运动。
    数据必须过滤掉转场、静态、低美学、文字和 caption mismatch。

Stage III:
    最后用高质量数据把模型对齐到高分辨率输出。
```

用迁移理解：

```text
text-to-video:
    text controls content and motion.

image-to-video:
    image fixes first-frame identity and appearance.

frame interpolation:
    endpoints fix start/end, model fills plausible motion.

multi-view:
    object identity fixed, camera path becomes a special kind of motion.

camera LoRA:
    small temporal adapters steer learned motion prior.
```

如果只记一个结论：

```text
SVD 的强点不是“会生成视频”。
它的强点是把视频预训练得到的 motion prior 变成一个可迁移基础模型。
```

## 16. 盲点与后续阅读

最大盲点是评估。论文大量依赖 human preference 和 Elo ranking，这对视频质量很重要，但也意味着结果受 prompt 集合、标注界面、采样设置、闭源 baseline 获取方式影响。自动指标如 FVD、LPIPS、PSNR、CLIP-S 只能覆盖部分维度，不能完整衡量视频的时间一致性、物理合理性和语义可控性。

主要限制：

```text
短视频强，长视频仍困难。

一次生成多帧在训练和推理上都昂贵。

生成结果有时运动不足。

扩散采样慢，VRAM 要求高。

数据策展降低坏模式，但不能消除数据偏见和滥用风险。
```

后续阅读建议：

1. 回读 `High-Resolution Image Synthesis with Latent Diffusion Models`，理解 latent space 为什么让高分辨率生成可行。
2. 回读 `Elucidating the Design Space of Diffusion-Based Generative Models`，理解 SVD 使用的 EDM preconditioning 和噪声分布迁移。
3. 阅读 `Align your Latents: High-Resolution Video Synthesis with Latent Diffusion Models`，对比 SVD 继承的 Video LDM 架构。
4. 阅读 `Video Diffusion Models`、`Imagen Video` 或 `Make-A-Video`，对比 cascaded video diffusion 与 latent video diffusion 的取舍。
5. 阅读 `Zero123`、`SyncDreamer`，理解 SVD-MV 为什么能把视频先验迁移到多视角生成。
