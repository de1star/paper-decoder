## 1. 阅读计划回顾

这篇论文不要先读成“Stable Diffusion 也能做视频”。更好的入口是：

```text
图像 LDM 已经会生成高质量单帧
  -> 但逐帧采样会得到互不相关的图片
  -> 视频生成真正缺的是时序对齐
  -> 所以保留图像模型的空间能力
  -> 只插入并训练 temporal layers
  -> 再把 decoder、插帧模型、上采样模型也做 temporal alignment
  -> 得到高分辨率、长时间、可文本控制的视频生成系统
```

建议按以下顺序阅读：

1. `Abstract`、`Introduction`、Fig. 2、Fig. 3：确认论文的主张是“把强图像 LDM 对齐成视频 LDM”，不是从零训练视频模型。
2. `Section 2`：复习 LDM 的扩散目标，重点看 `x_tau = alpha_tau x + sigma_tau epsilon` 和 `epsilon / v` 预测。
3. `Section 3.1`：精读 temporal layer 如何插入空间 UNet，以及为什么冻结 spatial layers。
4. `Section 3.1.1`：读 temporal decoder finetuning。这里解释为什么 latent 已经一致，decode 后仍可能闪烁。
5. `Section 3.2-3.3`：读 prediction model 和 interpolation model，理解长视频与高帧率怎么拆开解决。
6. `Section 3.4`：读 temporal SR model，理解高空间分辨率为什么交给局部视频上采样器。
7. `Section 4` 和 `Appendix G-H`：把实验当成因果证据，特别关注 pixel baseline、end-to-end LDM、attention-only、video upsampler、decoder finetuning。
8. `Appendix B-D`：回到局限、伦理风险，以及 “convolutional in time / space” 的泛化条件。

## 2. 阅读主线

显式问题是：

```text
如何把 latent diffusion model 扩展到高分辨率视频生成，
同时避免视频扩散模型在训练成本、显存、数据规模和长时一致性上的爆炸？
```

真正的结构张力是：

```text
视频生成看起来需要一个完整的 spatiotemporal diffusion model。
但高质量视频包含两个不同问题：

1. 单帧视觉质量：物体、纹理、构图、文字条件、风格表达。
2. 时序一致性：相邻帧运动连续、身份保持、长时间方向稳定、上采样不闪烁。

如果从零训练一个视频模型，会同时承担这两个问题，成本极高。
如果直接逐帧使用图像模型，单帧质量很好，但每一帧独立采样，视频不成立。
```

第一原则是：

```text
不要重新学习图像世界。
先继承图像 LDM 的空间生成能力，
再只学习“如何让这些图像 latent 在时间维度上对齐”。
```

所以 Video LDM 的核心判断是：

```text
spatial prior 来自大规模图像 LDM；
motion / temporal alignment 由少量新增 temporal layers 学习；
高分辨率由 latent generation + temporal interpolation + video-aware SR 组成。
```

## 3. 论文要解决的问题

在这篇论文之前，视频扩散模型常见路线是直接在像素空间或低分辨率视频上建模。问题是：

```text
像素空间视频张量:
    T x 3 x H x W

如果 H、W 很大，T 又不短，
每一步 diffusion 都要处理完整时空张量，
训练和采样成本会迅速超过图像扩散模型。
```

LDM 已经给图像生成提供了一个解决方式：

```text
图像 x
  -> encoder E
latent z = E(x)
  -> diffusion in latent space
latent z_0
  -> decoder D
图像 x_tilde = D(z_0)
```

Video LDM 的问题重构是：

```text
能不能把视频生成的大部分成本也移动到 latent space，
并且最大化复用已经训练好的 image LDM？
```

论文把答案拆成五个部件：

1. `temporal layers`：插入图像 LDM 的 UNet 中，只训练这些层来对齐帧。
2. `temporal decoder finetuning`：让 image autoencoder 的 decoder 输出更稳定的视频帧。
3. `prediction model`：给定上下文帧，递推生成更长视频。
4. `interpolation model`：在关键帧之间补帧，提高 fps。
5. `temporal SR model`：把低分辨率视频局部上采样到高分辨率，并保持时间一致。

## 4. 核心贡献

这篇论文的贡献可以读成四层。

第一层是训练策略。论文不是从零训练视频扩散模型，而是把预训练 image LDM 变成 video LDM：冻结空间层，插入 temporal attention 和 3D convolution residual blocks，只训练新增时序层。

第二层是完整视频生成栈。论文把高分辨率视频拆成 sparse keyframe generation、latent frame interpolation、decode、video upsampling，而不是要求一个模型一次解决所有空间和时间尺度。

第三层是实际任务验证。论文在 512 x 1024 的真实驾驶视频生成上超过 Long Video GAN，并能生成多分钟长视频；同时把 Stable Diffusion 转成 text-to-video 模型，可生成最高 1280 x 2048 的视频。

第四层是 temporal layer 的可迁移性。Stable Diffusion 上训练出的 temporal layers 可以插到 DreamBooth 微调后的图像模型中，实现 personalized text-to-video。

## 5. 核心符号

```text
x: 数据样本。对视频来说，x in R^{T x 3 x H_tilde x W_tilde}

T: 视频帧数
H_tilde, W_tilde: 原始像素空间高度和宽度

E: LDM autoencoder 的 encoder，逐帧编码视频
D: LDM autoencoder 的 decoder，把 latent 解码回像素帧

z = E(x): 视频 latent，z in R^{T x C x H x W}
C: latent channel 数
H, W: latent 空间高度和宽度

tau: diffusion time / timestep
alpha_tau, sigma_tau: 噪声日程中的信号系数和噪声系数
epsilon: 标准高斯噪声，epsilon ~ N(0, I)

x_tau: 加噪后的样本，x_tau = alpha_tau x + sigma_tau epsilon
z_tau: 加噪后的 latent 视频

f_theta: 原始 image LDM 的 denoiser / UNet
theta: image LDM 空间层参数
phi: 新增 temporal layers 的参数
f_{theta, phi}: 插入 temporal layers 后的视频 denoiser

l^i_theta: 第 i 个 spatial layer
l^i_phi: 第 i 个 temporal layer
alpha^i_phi: temporal layer 输出与 spatial output 合并时的可学习 mixing parameter

c: 条件信息，例如 text prompt、day/night label、crowdedness label

m_S: temporal binary mask，用于标记哪些上下文帧已知、哪些帧需要预测或插值
S: 上下文帧数量
c_S = (m_S o z, m_S): mask 后的 latent frames 与 mask 本身组成的空间条件

g_{theta, phi}: video-aware super-resolution diffusion model
tau_gamma: 给低分辨率 conditioning 加噪时使用的噪声等级
s: classifier-free guidance / context guidance scale
```

和 LDM / DDPM 的对应关系：

```text
DDPM:
    x_0 -> x_t -> epsilon_theta(x_t, t)

LDM:
    x -> z = E(x)
    z_0 -> z_t -> epsilon_theta(z_t, t)

Video LDM:
    video x -> framewise z = E(x)
    z_0^{1:T} -> z_t^{1:T}
    f_{theta, phi}(z_t^{1:T}, t, condition)
```

关键变化不是扩散目标被推翻，而是 denoiser 的结构从“空间模型”变成“空间层 + 时间层”。

## 6. 方法框架

Video LDM 的整体系统可以读成一个分层流水线：

```text
1. 训练或复用 image LDM
   输入: 单帧图像
   学到: 高质量 spatial prior

2. 插入 temporal layers
   输入: encoded video latent
   学到: frame alignment / motion consistency

3. temporal decoder finetuning
   输入: video latent
   目标: decode 后像素帧不闪烁

4. prediction model
   输入: 0/1/2 个上下文帧
   目标: 生成后续 sparse keyframes

5. interpolation model
   输入: 两端关键帧
   目标: 在中间补帧，提高 fps

6. video SR model
   输入: 低分辨率视频
   目标: 局部上采样，同时保持时间一致
```

论文的架构图 Fig. 5 可以压缩成：

```text
Key Frame LDM
  -> Latent Frame Interpolation I
  -> Latent Frame Interpolation II
  -> LDM Decoder
  -> Video Upsampler
```

这个拆法的关键是：长时语义变化在低 fps / latent space 中建模；高 fps 由插帧补齐；高像素分辨率由局部上采样器补齐。

## 7. 关键公式与推导

### 7.1 扩散训练目标

论文从标准扩散目标开始：

```text
x_tau = alpha_tau x + sigma_tau epsilon
epsilon ~ N(0, I)
```

训练目标是：

```text
E_{x ~ p_data, tau ~ p_tau, epsilon ~ N(0,I)}
    || y - f_theta(x_tau; c, tau) ||_2^2
```

其中 `y` 可以是：

```text
epsilon: 直接预测噪声
v = alpha_tau epsilon - sigma_tau x: v-prediction
```

### 推导

从干净样本 `x` 构造带噪样本：

```text
x_tau = alpha_tau x + sigma_tau epsilon
```

这一步把 diffusion time `tau` 映射成一个信噪比状态。`tau` 越大，`sigma_tau` 通常越大，样本越接近纯噪声。

模型训练时看到的是 `x_tau`、条件 `c` 和时间 `tau`，它要预测一个目标 `y`。如果 `y = epsilon`，模型学习“这张带噪图里加了什么噪声”；如果 `y = v`，模型学习一种信号与噪声的混合参数化。

### 直觉

扩散训练的本质不是让模型直接生成视频，而是让模型学会：

```text
给定一个噪声强度下的混合样本，
判断它应该往哪个方向去噪。
```

Video LDM 沿用这个目标，只是把样本从单图像扩展成视频 latent 序列。

### 7.2 从图像 LDM 到视频 LDM

原始 image LDM 有一组空间层：

```text
l^1_theta, l^2_theta, ..., l^L_theta
```

Video LDM 在这些层之间插入时间层：

```text
l^1_theta -> l^1_phi -> l^2_theta -> l^2_phi -> ... -> l^L_theta -> l^L_phi
```

视频 latent 的形状是：

```text
z in R^{T x C x H x W}
```

训练时，空间层把视频帧当成 batch 中独立图片处理：

```text
(B, T, C, H, W) -> (B*T, C, H, W)
```

到了 temporal layer，再 reshape 回视频：

```text
z' <- rearrange(z, (b t) c h w -> b c t h w)
z' <- l^i_phi(z', c)
z' <- rearrange(z', b c t h w -> (b t) c h w)
```

temporal layer 的输出与原空间输出合并：

```text
output = alpha^i_phi z + (1 - alpha^i_phi) z'
```

### 推导

这条公式可以拆成三个对象：

```text
z: image LDM spatial layer 的输出
z': temporal layer 处理完整视频后的输出
alpha^i_phi: 控制保留图像模型能力与引入时序修正的比例
```

当：

```text
alpha^i_phi = 1
```

则：

```text
output = z
```

模型退化回原来的 image LDM。也就是说，跳过 temporal layers 后，原图像生成能力仍然存在。

当：

```text
alpha^i_phi < 1
```

temporal layer 开始影响输出，模型在保留 spatial prior 的同时学习帧间关系。

### 直觉

这像给强图像模型加一个“时间校正器”。空间层负责回答“每帧应该长什么样”，时间层负责回答“这些帧如何组成同一个运动过程”。

这个设计比端到端训练视频模型更保守，也更有效率，因为它不让有限的视频数据重新学习图像生成能力。

### 7.3 temporal layer 训练目标

Video LDM 的训练目标是：

```text
arg min_phi E_{x ~ p_data, tau ~ p_tau, epsilon ~ N(0,I)}
    || y - f_{theta, phi}(z_tau; c, tau) ||_2^2
```

注意优化变量只有 `phi`。

```text
theta: frozen spatial layers
phi: trainable temporal layers
```

### 推导

视频帧先逐帧编码：

```text
z = E(x)
```

再对 latent video 加噪得到：

```text
z_tau
```

模型输入 `z_tau`，预测 `y`。因为 `theta` 冻结，梯度只更新 temporal layers。

### 直觉

冻结 spatial layers 是这篇论文的关键归纳偏置：

```text
不要让视频数据破坏已经学好的图像分布；
只让视频数据告诉模型如何在时间上组织这些图像。
```

实验里 end-to-end LDM 的表现明显更差，说明这个归纳偏置不是装饰，而是核心。

### 7.4 prediction model：长视频生成

短视频可以一次生成，但长视频不能无限增加 `T`。论文用上下文帧递推预测：

```text
c_S = (m_S o z, m_S)
```

其中：

```text
m_S: mask，标出哪些帧已知
m_S o z: 只保留上下文帧的 latent
c_S: mask 后的 latent 与 mask 本身拼接成条件
```

训练目标变成：

```text
E_{x ~ p_data, m_S ~ p_S, tau ~ p_tau, epsilon}
    || y - f_{theta, phi}(z_tau; c_S, c, tau) ||_2^2
```

论文实际训练条件可以是：

```text
S = 0: 无上下文，生成初始视频段
S = 1: 给定 1 帧，预测后续
S = 2: 给定 2 帧，捕捉运动方向后预测后续
```

### 推导

长视频生成时不是一次生成全部帧，而是：

```text
先生成一个起始帧或短片段
  -> 取最新 1/2 帧作为 context
  -> 预测下一段 keyframes
  -> 把新预测帧加入历史
  -> 重复
```

两帧上下文比一帧更有用，因为两帧能提供运动方向。

### 直觉

prediction model 把长视频问题从：

```text
一次性生成一个巨大 T 的视频张量
```

改成：

```text
在局部时间窗口里反复预测未来。
```

这降低显存压力，也让模型更容易保持短期一致性。

### 7.5 context guidance

论文对上下文条件使用 classifier-free guidance 形式：

```text
f'_{theta, phi}(z_tau; c_S)
  = f_{theta, phi}(z_tau)
    + s * (f_{theta, phi}(z_tau; c_S) - f_{theta, phi}(z_tau))
```

其中 `s >= 1` 是 guidance scale。

### 推导

括号中的差值：

```text
f(z_tau; c_S) - f(z_tau)
```

表示“上下文条件让模型输出发生的方向性变化”。乘以 `s` 后，这个条件方向被放大。

### 直觉

context guidance 强化“遵守上下文帧”的倾向。实验里它降低 FVD，但会牺牲一些单帧视觉质量，表现为 FID 变差。

### 7.6 temporal interpolation：高帧率生成

高分辨率视频不只是空间分辨率高，还需要高 fps。论文不直接用主模型生成所有帧，而是：

```text
先生成低 fps keyframes
再训练 interpolation model 补中间帧
```

插帧也复用 mask conditioning：

```text
已知帧: keyframes
mask 掉的帧: 要插出来的中间帧
```

论文训练一个 `T -> 4T` 插帧模型，并可连续使用两次：

```text
1.875 fps -> 7.5 fps
7.5 fps -> 30 fps
```

### 直觉

关键帧模型负责语义跨度，插帧模型负责局部运动平滑。这样避免让一个模型同时负责“大语义变化”和“密集帧间连续性”。

### 7.7 temporal SR model：高空间分辨率

论文用扩散上采样器把视频放大 4 倍。上采样训练目标是：

```text
E_{x ~ p_data, (tau, tau_gamma) ~ p_tau, epsilon ~ N(0,I)}
    || y - g_{theta, phi}(x_tau; c_{tau_gamma}, tau_gamma, tau) ||_2^2
```

低分辨率 conditioning 会被加噪：

```text
c_{tau_gamma} = alpha_{tau_gamma} x + sigma_{tau_gamma} epsilon
```

### 推导

上采样器输入两类信息：

```text
x_tau: 当前要去噪的高分辨率样本
c_{tau_gamma}: 加噪后的低分辨率条件图 / 视频
```

`tau_gamma` 告诉模型低分辨率条件本身有多 noisy。这样上采样器不会过度依赖一个可能带噪或有伪影的低分辨率输入。

### 直觉

如果逐帧使用 image upsampler，每帧可能都很清晰，但细节会闪烁。Video LDM 的做法是给 upsampler 也插入 temporal layers，让它在局部视频块上保持一致。

## 8. 训练过程

### 8.1 基础 Video LDM 训练

```text
repeat:
    sample a video x from video dataset
    encode each frame with frozen encoder E
    obtain latent video z = E(x)
    sample diffusion timestep tau
    sample noise epsilon
    construct z_tau
    pass z_tau through frozen spatial layers theta and trainable temporal layers phi
    predict epsilon or v
    compute denoising MSE loss
    update only phi
```

训练时已知：

```text
完整干净视频 x
完整 latent video z
噪声 epsilon
条件 c
目标 y
```

训练时要学：

```text
在每个噪声等级下，如何利用时间维度让帧间关系一致。
```

### 8.2 temporal decoder finetuning

```text
repeat:
    sample a real video x
    encode frames independently with frozen image encoder E
    decode latent sequence with decoder D plus temporal layers
    compare reconstructed video with original video
    use video-aware discriminator based on 3D convolutions
    update decoder temporal components
```

这个步骤解决的问题是：

```text
即使 latent 序列时间上一致，
image-only decoder 逐帧解码也可能产生像素级闪烁。
```

论文保持 encoder 不变，因为 latent diffusion model 依赖这个 encoder 产生的 latent 空间。只调整 decoder，可以让视频输出更稳定，同时不破坏主 LDM 的 latent 输入定义。

### 8.3 长视频预测训练

```text
repeat:
    sample a video x
    encode to latent z
    sample S in {0, 1, 2}
    build temporal mask m_S
    build context condition c_S = (m_S o z, m_S)
    add diffusion noise to z
    predict denoising target with f_{theta, phi}(z_tau; c_S, c, tau)
    update temporal layers
```

训练与推理的区别：

```text
training:
    上下文帧和未来帧都来自真实视频，只是未来帧被 mask。

inference:
    上下文帧可能来自模型上一轮生成结果，误差会递推累积。
```

所以论文用 context guidance 稳定推理过程。

### 8.4 插帧训练

```text
repeat:
    sample high-fps video
    choose keyframes as known frames
    mask intermediate frames
    condition on keyframes and mask
    train Video LDM to denoise / reconstruct missing frames
```

插帧模型与预测模型共用思想，但任务方向不同：

```text
prediction:
    已知前面的帧，预测未来。

interpolation:
    已知两端关键帧，补中间。
```

### 8.5 video upsampler 训练

```text
repeat:
    sample high-resolution video patch
    create low-resolution conditioning video
    optionally add noise augmentation to low-resolution condition
    add diffusion noise to high-resolution target
    use video-aware SR diffusion model to predict denoising target
    update temporal SR layers
```

论文强调上采样器可以 patch-wise 训练，因为它不需要建模长程运动，只需要在低分辨率视频条件下补局部细节。

## 9. 推理 / 采样过程

### 9.1 驾驶视频生成

```text
generate an initial frame with image LDM

for each segment:
    use prediction model with 1 or 2 context frames
    generate sparse keyframes
    append generated keyframes to video

optionally:
    interpolate keyframes from low fps to 7.5 fps
    interpolate again from 7.5 fps to 30 fps
    decode latent video to pixel space
    apply video-aware 4x upsampler on local frame windows
```

论文在 RDS 驾驶数据上验证可生成最多 5 分钟的视频。

### 9.2 Stable Diffusion text-to-video

```text
start from public Stable Diffusion checkpoint
briefly finetune spatial layers on WebVid frames
insert temporal layers
train temporal alignment on WebVid text-video pairs
sample latent video conditioned on text prompt
decode frames
optionally apply temporal interpolation
optionally apply video-finetuned Stable Diffusion 4x upscaler
```

输出规模：

```text
base training resolution: 320 x 512
generated sequence: 113 frames
rendering example:
    4.7 seconds at 24 fps
    3.8 seconds at 30 fps
with upsampler:
    up to 1280 x 2048
```

### 9.3 personalized text-to-video

```text
train DreamBooth on a small image set
obtain personalized image LDM spatial backbone
insert previously trained temporal layers
generate video using rare token such as "sks"
```

关键结论：

```text
temporal layers are transferable.
```

也就是说，时间层不是只记住某个固定图像模型的输出分布，而是学到了一种可迁移的时序对齐能力。

## 10. 实验结论

### 10.1 RDS 高分辨率驾驶视频

数据集：

```text
RDS: 683,060 个真实驾驶视频
长度: 8 秒
分辨率: 512 x 1024
帧率: 最高 30 fps
条件: day/night、crowdedness
```

主结果在 128 x 256 分辨率上比较 Long Video GAN：

```text
LVG:

    FID 53.5

    FVD 389
    FID 31.6

Video LDM (conditional):
    FVD 356
    FID 51.9
```

因果解释：

```text
Video LDM 相比 LVG 有更好的视频分布相似性；
加入条件能进一步降低 FVD，但 FID 不一定同步改善。
```

人类评估：

```text

    62.03% 偏好 Ours
    31.65% 偏好 LVG
    6.33% 认为相同
```

这说明定量 FVD/FID 之外，人类也更偏好 Video LDM 的真实感。

### 10.2 消融实验

论文比较了几种关键替代方案：

```text
Pixel-baseline:
    FVD 639.56
    FID 59.70

End-to-end LDM:
    FVD 1155.10
    FID 71.26

Attention-only:
    FVD 704.41
    FID 50.01

Ours:
    FVD 534.17
    FID 48.26

Ours context-guided:
    FVD 508.82
    FID 54.16
```

这些消融支持四个判断：

1. latent space 比 pixel space 更适合高分辨率视频扩散训练。
2. 直接 end-to-end 从视频训练 LDM 很差，说明 image pretraining / spatial prior 很关键。
3. 只用 temporal attention 不够，3D convolution residual blocks 对上下文帧空间条件很重要。
4. context guidance 能改善 FVD，但可能损伤 FID。

## 11. 与 LDM / SVD 的关系

和 LDM 的关系：

```text
LDM 解决的是:
    高分辨率图像扩散太贵

Video LDM 继承这个答案:
    高分辨率视频扩散也应放到 latent space
```

Video LDM 没有改变 LDM 的核心范式，而是把 `z in R^{C x H x W}` 扩展为：

```text
z in R^{T x C x H x W}
```

再通过 temporal layers 处理 `T` 维。

和后来的 Stable Video Diffusion 的关系可以这样理解：

```text
Video LDM:
    重点是提出从 image LDM 到 video LDM 的 temporal alignment 机制，
    并验证驾驶视频、text-to-video、DreamBooth 迁移。

Stable Video Diffusion:
    进一步系统研究大规模视频数据策展、训练阶段和泛化任务，
    更强调数据分布与训练 recipe。
```

所以阅读顺序上，这篇更适合回答：

```text
图像 LDM 如何被结构性改造成视频 LDM？
```

SVD 更适合回答：

```text
如何把视频 LDM 扩展到更强、更通用的视频基础模型？
```

## 12. 本次疑问记录

### 问题 1：为什么不直接从零训练视频 LDM？

因为视频数据相比图像数据更少、更贵、更难清洗。从零训练会让模型同时学习空间视觉质量和时间一致性。论文实验中 end-to-end LDM 的 FVD / FID 都很差，说明这种方式既浪费图像先验，也更难优化。

### 问题 2：为什么 spatial layers 要冻结？

冻结 spatial layers 的目的不是省参数，而是保护图像模型已经学到的强空间先验。Temporal layers 只学习帧间对齐，任务更窄，所需视频数据更少。

### 问题 3：为什么 decoder 也要 video finetune？

LDM 的 latent diffusion 可以生成时间一致的 latent，但 image-only decoder 是逐帧训练的。即使 latent 平滑，decoder 的局部纹理重建也可能帧间抖动。Temporal decoder finetuning 直接对像素时序一致性负责。

### 问题 4：为什么插帧和预测要分开？

预测模型解决“未来会发生什么”，插帧模型解决“两帧之间如何平滑过渡”。前者偏语义和运动方向，后者偏局部连续性。如果让一个模型同时承担所有帧率和长时变化，显存和学习难度都会上升。

### 问题 5：为什么 FID 和 FVD 经常方向不一致？

FID 看单帧图像分布，FVD 看视频特征分布。逐帧 image upsampler 可以有好 FID，但因为细节闪烁，FVD 很差。Video LDM 的实验反复说明：视频生成不能只用单帧质量指标判断。

## 13. 最小心智模型

把这篇论文压缩到一个模型：

```text
Video LDM = Image LDM + Temporal Alignment Stack

Image LDM:
    提供每一帧的高质量空间生成能力

Temporal layers:
    插入 UNet，学习 latent 帧之间如何对齐

Temporal decoder:
    保证 latent decode 到 pixel 后不闪烁

Prediction model:
    用上下文帧递推生成长视频

Interpolation model:
    在关键帧之间补帧，提高 fps

Video SR model:
    在局部窗口中上采样，补高分辨率细节且保持一致
```

一句话版本：

```text
这篇论文的核心不是“训练一个视频模型”，
而是“把强图像 LDM 的 latent 空间沿时间维度对齐”。
```

## 14. 盲点与后续阅读

最重要的盲点是：temporal alignment 依赖强 image prior，因此它继承了图像模型的能力，也继承了图像模型的数据偏差和语义局限。它不是从视频数据中完整学习世界动力学，而是在已有图像生成能力之上学习局部时序一致性。

几个具体风险：

1. 长视频递推会累积误差。Appendix D 提到 convolutional-in-time 可以生成更长视频，但比较脆弱，论文更推荐 prediction model。
2. 单帧质量和时序一致性有 trade-off。Decoder finetuning 和 context guidance 都可能改善 FVD，但不一定改善 FID。
3. FVD 本身不稳定。Mountain Biking 实验中，Video LDM 有更好 FID 和人类偏好，但 FVD 不如 LVG，说明 FVD 可能偏好短期平滑而非真实细节。
4. WebVid 质量会限制 text-to-video。论文提到先在 WebVid 帧上微调 Stable Diffusion 会缓解分辨率 / 分布不匹配，但也可能伤害原始 SD 的图像质量。
5. 伦理和数据许可问题没有被技术方案解决。论文明确指出模型不适合直接产品化，未来需要使用伦理来源和商业可用的数据训练。

后续阅读建议：

```text
LDM:
    理解 latent space 为什么降低高分辨率扩散成本。

EDM:
    理解扩散采样、预条件化、噪声分布这些可拆设计。

Stable Video Diffusion:
    理解 Video LDM 思路如何进一步扩展到大规模 curated video dataset 和通用视频基础模型。
```
